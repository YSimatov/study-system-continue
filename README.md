## 2. Реализованные функции

### 2.1 Аутентификация и роли

| Роль | Права |
|---|---|
| **STUDENT** | Просмотр теории, прохождение тестов и экзаменов, личный кабинет, удаление своего аккаунта |
| **TEACHER** | Всё что STUDENT + управление темами, вопросами, просмотр профилей студентов |
| **ADMIN** | Всё что TEACHER + управление пользователями, смена ролей, удаление пользователей |

**Ключевые правила безопасности (server-side):**
- Администратор не может удалить другого администратора
- Администратор не может изменить роль другому администратору
- Страница результатов теста защищена: студент видит только свои попытки

### 2.2 Учебный процесс

#### Подтемы — вкладочный интерфейс
Страница подтемы `(/topics/[slug]/[subtopicSlug])` содержит две вкладки, переключаемые через URL `?tab=`:

- **Теория** (`?tab=theory`) — Markdown-контент по подтеме
- **Тест самоконтроля** (`?tab=quiz`) — Вопросы с вариантами ответов, **с подсветкой** правильных/неправильных ответов после завершения

#### Экзамен по теме
- Путь: `/exam/[topicSlug]`
- Рандомная подборка вопросов из всех подтем темы
- **Без подсветки** правильных ответов во время и после прохождения (требование)
- Результат сохраняется в модель `ExamAttempt`

#### Страница результатов теста
- Путь: `/result/[attemptId]`
- Показывает каждый вопрос с цветовой маркировкой:
  - 🟢 Зелёная граница — правильный ответ
  - 🔴 Красная граница — неправильный ответ
  - Бейдж «Правильный ответ» на верном варианте если выбрали не его

### 2.3 Личный кабинет студента

- Статистика: количество тестов, средний балл, экзамены
- Полная история экзаменов с цветовыми бейджами результата
- История тестов самоконтроля с разбивкой по подтемам
- **Удаление своего аккаунта** с двойным подтверждением (кнопка меняется и требует повторного нажатия)

### 2.4 Управление пользователями (ADMIN)

**Страница `/dashboard/users`:**
- Таблица всех пользователей с ролями, количеством попыток, датой регистрации
- Кнопка **«Просмотр профиля →»** — для просмотра истории любого пользователя
- Выпадающий список **«Изменить роль»** — мгновенное сохранение, защита от смены роли другому ADMIN
- Кнопка **«Удалить»** с двойным подтверждением — нельзя удалить других ADMIN

**Страница `/dashboard/users/[userId]`:**
- 4 карточки: тестов / средний балл тесты / экзаменов / средний балл экзамены
- История экзаменов с результатами и датами
- Тесты самоконтроля, сгруппированные по темам: правильно / ошибок / результат
- Кнопка **«Просмотр ответов →»** ведёт на `/result/[attemptId]` с полным разбором

### 2.5 Use Case диаграмма

- Путь: `/use-case`
- Интерактивный SVG-компонент в тёмном стиле (фон `#1a1a2e`)
- Ортогональные стрелки (L-образные) через вертикальные «шины»
- Цветовая схема по актору: 🟢 Учащийся · 🟡 Преподаватель · 🔴 Администратор · 🔵 Общие
- Наследование Admin → Teacher показано через пунктирную стрелку с открытым треугольником

## 3. База данных

### Схема Prisma (ключевые модели)

```prisma
User         — id, name, email, passwordHash, role (STUDENT|TEACHER|ADMIN)
Topic        — id, title, slug, description
Subtopic     — id, title, slug, contentMarkdown, topicId
Question     — id, text, subtopicId
Option       — id, text, isCorrect, questionId
Attempt      — id, score, total, userId, subtopicId        ← тест самоконтроля
AttemptAnswer— id, attemptId, questionId, selectedOptionId, isCorrect
ExamAttempt  — id, score, total, userId, topicId           ← экзамен
```

### Supabase / Подключение

Используется Transaction Pooler с `pgbouncer=true` для совместимости с Prisma:
```
DATABASE_URL=postgresql://...?pgbouncer=true&connection_limit=1
```

## 4. Серверные экшены (Server Actions)

| Файл | Экшен | Описание |
|---|---|---|
| `auth-actions.ts` | `registerAction` | Регистрация нового пользователя |
| `auth-actions.ts` | `loginAction` | Вход через credentials |
| `user-actions.ts` | `deleteMyAccountAction` | Удаление своего аккаунта + signOut |
| `user-actions.ts` | `deleteUserAction` | Удаление пользователя (ADMIN only, не ADMIN) |
| `user-actions.ts` | `changeRoleAction` | Смена роли (ADMIN only, не ADMIN) |
| `quiz-actions.ts` | `submitQuiz` | Сохранение попытки теста с ответами |
| `exam-actions.ts` | `submitExam` | Сохранение результата экзамена |
| `content-actions.ts` | — | CRUD тем и подтем |
| `question-actions.ts` | — | CRUD вопросов и вариантов |

## 5. Команды для запуска

```bash
# Установка зависимостей
npm install

# Применение миграций к БД
npx prisma migrate deploy

# Заполнение начальными данными
npx prisma db seed

# Запуск в режиме разработки
npm run dev

# Создание пользователей admin/teacher
```
